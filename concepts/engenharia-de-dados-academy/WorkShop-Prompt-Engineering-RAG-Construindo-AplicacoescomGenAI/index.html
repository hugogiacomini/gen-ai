🎯 Técnicas de Prompt Engineering

Chain of Thought (Cadeia de Pensamento):
"Resolva passo a passo: Qual é 15% de 240?
1. Primeiro, converta 15% para decimal: 0.15
2. Multiplique: 240 × 0.15
3. Resultado: 36"
Few-Shot Learning:
"Classifique o sentimento:
Exemplo 1: 'Adorei o produto!' → Positivo
Exemplo 2: 'Terrível experiência' → Negativo
Agora classifique: 'O atendimento foi excepcional'"

🔍 RAG - Retrieval Augmented Generation

🎯 O que é RAG?

RAG (Retrieval Augmented Generation) é uma técnica que combina a capacidade generativa dos LLMs com a recuperação de informações específicas de bases de dados externas, permitindo respostas mais precisas e atualizadas.

🔄 O processo offline prepara os dados, enquanto o processo online responde às consultas em tempo real

🔧 Componentes Principais do RAG

✂️ Chunking (Segmentação)

Processo de dividir documentos grandes em pedaços menores e gerenciáveis. Estratégias comuns:

  • 📏 Fixed-size chunking: Tamanho fixo (ex: 512 tokens)
  • 📝 Semantic chunking: Baseado em semântica
  • 🏗️ Structural chunking: Por estrutura do documento
🔢 Embeddings

Representação vetorial de texto que captura significado semântico. Modelos populares:

  • 🤗 Sentence Transformers: all-MiniLM-L6-v2
  • 🎯 OpenAI: text-embedding-ada-002
  • ☁️ Cohere: embed-multilingual-v2.0

⚖️ Trade-offs Importantes

Ao escolher componentes para RAG, considere:

  • 🎯 Precisão vs Velocidade: Modelos maiores são mais precisos mas mais lentos
  • 💰 Custo vs Performance: APIs pagas vs modelos self-hosted
  • 🌐 Latência vs Qualidade: Busca aproximada vs busca exata
  • 📊 Tamanho do Chunk vs Contexto: Chunks menores perdem contexto

📈 Métricas de Avaliação RAG

RAGAS (RAG Assessment):
• Context Precision: Precisão do contexto recuperado
• Context Recall: Cobertura do contexto relevante
• Faithfulness: Fidelidade da resposta ao contexto
• Answer Relevancy: Relevância da resposta à pergunta
📚 Referência: RAGAS: Automated Evaluation of Retrieval Augmented Generation

⚠️ Desafios do LLMOps

Diferenças principais entre MLOps tradicional e LLMOps:

  • 💰 Custo: APIs pagas por token vs modelos self-hosted
  • ⏱️ Latência: Respostas podem demorar segundos
  • 🎯 Avaliação: Métricas qualitativas vs quantitativas
  • 📝 Prompt Management: Versionamento de prompts
  • 🔒 Segurança: Prompt injection, data leakage

📝 Exemplo Prático: RAG com Python

Setup Básico RAG:
import openai
from langchain.vectorstores import Chroma
from langchain.embeddings import OpenAIEmbeddings
from langchain.text_splitter import CharacterTextSplitter

# 1. Processar documentos
text_splitter = CharacterTextSplitter(
  chunk_size=1000,
  chunk_overlap=200
)
chunks = text_splitter.split_text(document)

# 2. Criar embeddings e armazenar
embeddings = OpenAIEmbeddings()
vectorstore = Chroma.from_texts(chunks, embeddings)

# 3. Buscar e gerar resposta
def query_rag(question):
  relevant_docs = vectorstore.similarity_search(question)
  context = "\n".join([doc.page_content for doc in relevant_docs])
  prompt = f"Context: {context}\n\nQuestion: {question}\nAnswer:"
  return openai.ChatCompletion.create(model="gpt-3.5-turbo", messages=[{"role": "user", "content": prompt}])

🔧 Otimizações Avançadas

⚡ Técnicas de Otimização
  • 🎯 Hybrid Search: Combinar busca semântica + keyword
  • 🔄 Re-ranking: Re-ordenar resultados com modelos especializados
  • 📝 Query Expansion: Expandir queries com sinônimos
  • 🎛️ Filtering: Aplicar filtros de metadata
  • 💾 Caching: Cache de embeddings e respostas

🚨 Problemas Comuns e Soluções

Problema: Respostas inconsistentes
Solução: Melhorar prompt engineering, usar temperature baixa
Problema: Contexto irrelevante recuperado
Solução: Ajustar chunking strategy, usar filtros de metadata
Problema: Latência alta
Solução: Implementar cache, usar modelos menores quando possível
Problema: Custos elevados
Solução: Otimizar tokens, usar modelos locais quando viável
